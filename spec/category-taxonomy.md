# ORS Category Taxonomy

Every ORS skill belongs to exactly **one** of 18 top-level categories. The category is part of the skill's path (`skills/<category>/<slug>/SKILL.md`) and is also recorded in the body metadata block.

## Ordering

Categories are ordered roughly by **research workflow stage**:

1. **Data acquisition** — `bioinformatics-sequence`, `chemoinformatics`, `lab-automation`
2. **Analysis** — `machine-learning-bio`, `image-analysis-microscopy`, `data-engineering`
3. **Discovery** — `literature-research`, `scientific-thinking`
4. **Communication** — `scientific-writing`, `scientific-communication`, `scientific-visualization`
5. **Evaluation** — `peer-review`
6. **Career & funding** — `research-grants`, `career-navigation`
7. **Cross-cutting** — `open-science`, `ethics-compliance`, `mentorship-teaching`, `humanizer-skills`

This ordering is informational. The `INDEX.md` is generated by script, not hand-ordered.

## Adding a new category

New categories are added via PR. The bar is high — categories are for **major research-workflow phases** that don't fit into any of the existing 18. A single skill that's awkward to classify does not justify a new category; it should go in the closest existing category.

To propose a new category, open an issue first with:

1. The proposed name (kebab-case, ASCII).
2. A 1-paragraph description of what's in it.
3. A list of 5+ candidate skills that would belong to it.
4. Why these don't fit into any existing category.

## Renaming or removing a category

This is a **breaking change** and requires a major version bump. The skill paths in many users' setups will need to be updated. Avoid this unless there's a strong reason.
